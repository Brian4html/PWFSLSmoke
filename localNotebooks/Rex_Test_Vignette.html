Basics

We’ll go over a some of the basic functionality of R Markdowns.

Imbedding and Running Code and Results

Let’s play with a few different ways to work with and display dim(iris). Note that for all code shown below, you’ll want to substitute the ’ characters with the diagonal tick mark (non-shift of “~” on the keyboard) which is the special character that RMarkdown recognizes as flagging R code.

Entering the following in R Markdown file

'''{r}
dim(iris)
'''

returns the following result (two lines):

dim(iris)
## [1] 150   5

Now let’s try something a little different:

'''{r eval=FALSE}
dim(iris)
'''

This code yields, but this time no result is shown since eval=FALSE.

dim(iris)

And finally, here’s the same code again, except this time we use echo=FALSE to hide the code itself, and instead only display the result.

## [1] 150   5

Note that I tried using the same code as above, while calling echo=FALSE and eval=FALSE, and R didn’t like it at all, which makes sense, since why would you ever want to include code that you didn’t want to show nor evaluate? Anyway…

Now for some fun with in-line code. Entering “Two plus two equals ‘r 2 + 2’” (but with the smaller diagonal single quotes) yields the following – note the part called by r in quotes was evaluated in R.

Two plus two equals 4.

Note that the part in the small quotes above was evaluated, and the output was included within the text itself.

If you want to add a hyperlink to the vignette, you can google how to do it, or you can just look at this line of code in which case I have just demonstrated how to do this.

This package is designed to be used with R (>= 3.1.0) and RStudio so make sure you have those installed first.

Users will want to install the devtools package to have access to latest versions of some packages that are not yet available on CRAN.

The following packages should be installed with devtools by typing the following at the RStudio console:

devtools::install_github('rstudio/dygraphs')
devtools::install_github('rstudio/leaflet')
devtools::install_github('mazamascience/MazamaSpatialUtils', build_vignettes=TRUE)
devtools::install_github('mazamascience/PWFSLSmoke', build_vignettes=TRUE)

Any work with spatial data, e.g. assigning counties or watersheds, will require installation of required spatial datasets. To get these datasets you should execute the following commands from a shell terminal:

mkdir ~/Data
cd ~/Data
curl -O http://smoke.airfire.org/RData/Spatial.tar.gz
tar -xzf Spatial.tar.gz
